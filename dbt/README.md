# github_health_analytics dbt Project

This dbt project is the transformation layer for GitHub Health Analytics on BigQuery.

## What this project now contains

1. Source contracts for raw ingestion tables in `gh_raw`.
2. Staging models:
   - `stg_github_pulls`
   - `stg_github_issues`
   - `stg_github_pr_reviews`
3. Intermediate lifecycle models:
   - `int_pr_reviews_agg`
   - `int_pr_lifecycle`
   - `int_issue_lifecycle`
4. Marts:
   - `fact_pr_cycle_time`
   - `fact_issue_aging`
   - `mart_repo_health_daily`
5. Snapshot:
   - `snp_pr_state_history`
6. Domain tests for lifecycle and date sanity.

## Airflow-only setup (GCP keyfile auth)

### 1) Prerequisites

1. Service account permissions:
   - BigQuery Job User
   - Read on `gh_raw`
   - Write on `gh_analytics`
2. Keyfile path in repo (gitignored):
   - `airflow_env/gcp-key.json`
3. `.env` values:
   - `GCP_PROJECT_ID=winged-journey-488904-q9`
   - `BQ_DATASET_RAW=gh_raw`
   - `BQ_DATASET_ANALYTICS=gh_analytics`
   - `GCS_BUCKET_RAW=...`
   - `GCS_BUCKET_ARTIFACTS=...`

### 2) Build and start Airflow with dbt

From `airflow_env/`:

```bash
docker compose build
docker compose up -d
```

The Airflow services now mount `../dbt:/opt/airflow/dbt` and set:

- `DBT_BQ_PROJECT`
- `DBT_BQ_DATASET`
- `DBT_BQ_KEYFILE`
- `DBT_PROFILES_DIR=/opt/airflow/dbt`
- `DBT_PROJECT_DIR=/opt/airflow/dbt`

### 3) Validate dbt in scheduler container

```bash
docker compose exec airflow-scheduler dbt --version
docker compose exec airflow-scheduler dbt debug --project-dir /opt/airflow/dbt --profiles-dir /opt/airflow/dbt
docker compose exec airflow-scheduler dbt parse --project-dir /opt/airflow/dbt --profiles-dir /opt/airflow/dbt
```

### 4) First-time dbt run

```bash
docker compose exec airflow-scheduler dbt deps --project-dir /opt/airflow/dbt --profiles-dir /opt/airflow/dbt
docker compose exec airflow-scheduler dbt run --select staging --project-dir /opt/airflow/dbt --profiles-dir /opt/airflow/dbt
docker compose exec airflow-scheduler dbt test --select staging --project-dir /opt/airflow/dbt --profiles-dir /opt/airflow/dbt
```

### 5) Run from Airflow DAG

1. Open Airflow UI.
2. Trigger DAG `github_dbt_transform_manual`.
3. It runs:
   - `dbt_deps`
   - `dbt_run`
   - `dbt_test`

### 6) Validate outputs in BigQuery

Confirm tables in `gh_analytics`:

- `fact_pr_cycle_time`
- `fact_issue_aging`
- `mart_repo_health_daily`

and snapshot table in the snapshot schema generated by dbt (`<target_schema>_snapshots` by default with the current macro).
